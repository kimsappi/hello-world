Missing X11 include directory

If ./configure complains about missing X11 include directory, you do not have the packages required to compile the library.

Run find / 2>/dev/null | grep -e 'Xlib.h' -e 'XShm.h'

For MinilibX to work you need to have 2 results similar to these. The path may be different as long as it's listed in ./configure.

/usr/include/X11/Xlib.h

/usr/include/X11/extensions/XShm.h

If you only have 1 or none, you need to find and install the packages that include those files.

On Ubuntu you can use apt-file search filename (with | grep dev to prune some unnecessary results).

Install the packages that look the most promising. In my case they were libx11-dev and libxext-dev.

Missing strlcpy

Now you probably want to run ./configure and it still won't work. That's because your C library probably doesn't have strlcpy.

Go find the source for strlcpy or roll your own. Include the prototype in mlx_int.h and the file in the Makefile and run ./configure
